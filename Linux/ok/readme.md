3. **Настройка сети операционной системы**
- Установите имя машины как user-1.
> sudo vim /etc/hostname - прописываю user-1 и сохраняю.
- Установите часовой пояс
![Новый часовой пояс](pictures/timeMoscow.png)
- Выведите имена сетевых интерфейсов с помощью консольной команды
![Вывод имен сетевых интерфейсов](pictures/addr.png)
> *lo* (loopback device) – виртуальный интерфейс, присутствующий по умолчанию в любом Linux. Он используется для отладки сетевых программ и запуска серверных приложений на локальной машине.
- Используйте консольную команду, чтобы получить IP-адрес устройства, с которым вы работаете, от DHCP-сервера
![IP-адрес устройства](pictures/addr_show.png)
> DHCP расшифровывается как **Dynamic Host Configuration Protocol** (Протокол динамической конфигурации хоста). Это протокол, который позволяет устройствам в сети автоматически получать IP-адреса и другие сетевые параметры, такие как маска подсети и шлюз по умолчанию, от сервера DHCP.
- Определить и отобразить внешний IP-адрес шлюза (ip) и внутренний IP-адрес шлюза, также известный как IP-адрес по умолчанию (gw)
![внутренний IP-адрес](pictures/grep.png)
> **via 10.0.2.2** — это IP-адрес шлюза (маршрутизатора) в локальной сети.
> **dev enp0s3** — это интерфейс, через который осуществляется подключение
> **proto dhcp** — указывает, что IP-адрес был назначен через DHCP.
> **metric 100** — это метрика маршрута (используется для определения предпочтительности маршрута).
![внешний IP-адрес](pictures/curl.png)
> **176.214.237.33** — это внешний IP-адрес шлюза, через который устройство подключается к интернету. Этот адрес видим для всех.
> **sudo reboot** - Перезагрузил виртуальную машину для применения настроек

5. **Использование команды sudo**
> Команда `sudo` позволяет пользователю временно выполнять команды с правами администратора (root) без необходимости входить в систему как root. 
> Это помогает:
>
>     Работать безопаснее, так как пользователи используют повышенные привилегии только при необходимости.
>
>     Вести учёт всех действий, выполняемых с помощью`sudo`, что делает процесс прозрачным и упрощает контроль.
>Использовал `sudo usermod -aG sudo user-1` и `groups user-1`, потом `su - user-1`
![Использование команды sudo](pictures/whoami.png)

8. **Установка и базовая настройка службы SSHD**

![Установите службу SSHd](pictures/ssh.png)
- Включить автозапуск `sudo systemctl enable ssh`
![Автозапуск](pictures/status_ssh.png)
- Перезагрузка системы `sudo reboot`
- Проверка настройки порта с помощью netstat
![Проверка](pictures/tan.png)
>Объяснение ключей `-tan`:
        *   `-t` — только TCP.
        *   `-a` — все соединения.
        *   `-n` — числовые адреса.
>Значение `0.0.0.0:2022`: Служба ожидает подключений на порту 2022 со всех сетевых интерфейсов.
